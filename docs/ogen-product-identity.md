# Ogen Product Identity

Product name: **Ogen**

Hebrew meaning: **עוגן** — anchor.

Ogen is the product name for the Secure A2A Gateway / enterprise AI agent control plane. The technical category remains Secure A2A Gateway, but public-facing copy should use **Ogen** as the product brand.

## Primary tagline

```text
The trust layer and secure runtime for enterprise AI agents.
```

This is the preferred unified tagline. It combines the two strongest positioning directions:

- The trust layer for enterprise AI agents
- Secure runtime for enterprise AI agents

## Short tagline options

```text
The trust layer for enterprise AI agents.
```

```text
Secure runtime for enterprise AI agents.
```

Use the unified tagline by default in README, demo, LinkedIn, and product copy. Use the shorter variants when space is limited.

## Positioning

Ogen anchors every enterprise AI agent action to:

- verified user identity
- local user directory authorization
- connector trust
- scoped runtime tokens
- Gateway policy
- approval / authorization boundaries
- safe audit proof
- future SOC and observability export

## Product framing

```text
AI can interpret.
Ogen decides what is allowed.
External agents execute only approved actions.
Audit proves what happened.
```

Product principle:

```text
AI can interpret. Ogen decides. Runtime executes only approved actions. Audit proves what happened.
```

AI interpretation is a signal. Ogen authorization is the decision.

Ogen does not infer safety from missing metadata.

Tenant policy can restrict. Ogen guardrails cannot be bypassed.

Ogen guardrails are immutable platform safety rules.

Ogen reserved rule identities cannot be redefined by tenant policy.

Ogen does not just block. It explains which guardrail or tenant policy made the decision.

Every Ogen decision explains the exact rule, guardrail, or default-deny path that produced it.

## Public copy direction

Ogen should sound like a product, not a protocol name.

Avoid leading with:

```text
Secure A2A Gateway
```

Prefer:

```text
Ogen — The trust layer and secure runtime for enterprise AI agents.
```

Technical descriptions can still say:

```text
Ogen is a Secure A2A Gateway for identity-bound, policy-governed enterprise AI agent execution.
```

## LinkedIn / demo wording

```text
I call it Ogen.

AI agents are starting to act inside real enterprise systems — Jira, ServiceNow, GitHub, Monday, internal APIs.

But before giving them keys to the building, every action needs to be anchored to identity, permissions, consent, policy, and audit.

Ogen is the trust layer and secure runtime for enterprise AI agents.
```

## Naming decision

Selected name: **Ogen**

Reason:

- Israeli / Hebrew connotation without sounding forced
- easy enough to pronounce in English
- directly maps to trust-anchor semantics
- fits identity, policy, authorization, audit, and runtime governance
- stronger as a product name than “Secure A2A Gateway”
